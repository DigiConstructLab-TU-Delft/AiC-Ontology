Abstract

The Agents in Construction (AiC) ontology is a lightweight semantic model for coordinating on-site production in construction. It represents construction agents (human workers, robots, and other autonomous hardware) as they operate on a site: the processes they carry out, the elements they modify, the resources they handle, and the operational modes and metrics used to assess their performance and impact. To express how agent activities evolve over time, AiC introduces meta-properties that specify when a statement holds (at an instant or over an interval), what dimension of information it represents (planned, simulated, or performed), and who generated it. The ontology emphasizes generic, trade-agnostic concepts to ensure broad applicability across project types and delivery methods.


The namespace for AiC terms is https://w3id.org/aic#

The suggested prefix for the AiC namespace is aic


The AiC ontology is a deliverable of the research project Smart Mobile Factory for Infrastructure Projects (SMF4INFRA).

Introduction back to ToC

Coordinating production on construction sites requires a shared, machine-readable representation of key operational facts: who is acting (agents), what they are working on (processes and elements), where work is taking place (zones), which resources are being used or affected (equipment and materials), and how well the work is being executed (modes and metrics). These facts are dynamic - changing as the project progresses - and must therefore be expressed with explicit temporal and contextual information.

Ontology modules

The AiC ontology is organized into three modules - Agent, Focus, and Resource - that together describe the dynamics of agent operation on construction sites.

Agent module

The Agent module models construction agents, their capabilities, their operational modes and their locations. It enables expressing who the agents are, what they are capable of doing, their mode or condition and where they are operating at any point in time.

Focus module

The Focus module describes what agents direct their effort toward - either construction processes or building elements. Processes are decomposed into work packages, activities, and tasks, and linked to the elements they target. The module also represents process states and process metrics, supporting coordination and progress analysis across a project.

Resource module

The Resource module represents the materials and equipment handled by agents. It links resources to their types, quantities, and locations, and distinguishes between consumable and reusable materials. The module supports describing material and equipment requirements (for operation planning) and what is used, consumed, or supplied by agents during operation.

Including metadata about AiC statements (temporal validity, planned vs. performed, creators, ...)

Metadata in AiC is modeled using the class aic:InformationID. An instance of aic:InformationID represents this particular piece of information about a statement and is linked to the statement it describes via the object property aic:reifiesStatement. The statement itself is represented using standard RDF 1.1 reification (an instance of rdf:Statement with rdf:subject, rdf:predicate, and rdf:object).

Each aic:InformationID can capture temporal validity via three properties that describe when the reified statement holds. These temporal properties describe the validity of the statement itself (as reified by rdf:Statement), not the time when the data was entered into the knowledge base (i.e. transaction time).

  • aic:validAt - an instant at which the statement is valid.
  • aic:validFrom - the start of an interval during which the statement is valid.
  • aic:validUntil - the end of that interval.

Beyond temporal validity, aic:InformationID can be used to attach additional contextual metadata to the same statement, such as:

  • aic:informationDimension - the dimension of the information (e.g., planned, simulated, or performed).
  • aic:generatedOn - the time at which the information was generated.
  • aic:generatedBy - the entity or system that created it.

This provides a unified mechanism for expressing provenance, data-quality categories, workflow stages, and similar annotations about a statement.

RDF 1.2 introduces triple terms and quoted triples - formerly known as RDF-star - which allow triples to be used directly as subjects or objects of other triples. This enables attaching metadata such as temporal validity or provenance directly to quoted triples, without using explicit reification structures.

When the AiC ontology was designed, RDF 1.2 was still a W3C Working Draft and support for triple terms and SPARQL 1.2 quoted triple syntax was not yet consistent across RDF stores, SPARQL engines, and OWL reasoners. To ensure interoperability with existing RDF 1.1 tools, AiC therefore provides the aic:reifiesStatement object property to model bespoke RDF 1.1 reification.

In tools that fully support RDF 1.2 and SPARQL 1.2, the reification structures used in AiC can be systematically mapped to quoted triples (RDF-star syntax). In that case, a aic:InformationID instance refers to a triple term using the rdf:reifies object property. This yields more concise data and often simpler SPARQL queries. The two representations are intended to be semantically equivalent and can be transformed into one another where tooling permits.
